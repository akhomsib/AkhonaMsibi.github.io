Identifying Data Mapping Patterns

Tools: Python, SQL

This project, in collaboration with a global master data management firm, involved identifying data mapping patterns between customer-specific schema to increase efficiency of the mapping process for their data syndication platform. This meant handling over a million data points of unstructured data, highlighting some of the challenges in practical analytics projects. Multiple goal-driven solutions were developed, with text vectorization techniques and K-Means clustering algorithms utilized to effectively group customer schema for enhanced data synchronization.

Additionally, several classification models; including Support Vector Machine, XGBoost, and neural network models; as well as engineered prompts input into trained large language models were also evaluated to optimize the efficiency of schema matching by projecting mapping pairings, achieving up to 94% accuracy in prediction.

Achievements

  • Cleansed over 1 million data points of unstructured existing master schema and mapping files for global master data management firm, handling sparsity and formatting inaccuracies, in preparation for analytical modelling.
  • Boosted efficiency of data synchronization, utilizing Doc2vec text vectorization techniques and K-Means clustering algorithms to group customer-specific schema by identifying underlying structural patterns.
  • Optimized schema matching by evaluating classification models; including Support Vector Machine, XGBoost, and neural network models; to project mapping pairings, achieving 89% accuracy in mapping prediction.
  • Automated mapping generation for data syndication platform, engineering scripted prompts for input into trained LLM models to conclude data mappings based on attributes, obtaining predictive accuracy of 94%.
